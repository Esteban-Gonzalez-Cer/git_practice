## Descripción del proyecto
Breve pagina .html sin estilos, donde cada uno de los integrantes añadío divs o apartados para luego ser subidos a un repositiorio global usando la herramienta de control de versiones de Git.

## Instrucciones
Tener un repositorio donde todos los integrantes participen al añadir divs en el archivo html siguiendo las prácticas de ramas y control de versiones proporcionado por Git.

## Flujo
- Crear un repositorio en GitHub.
- Crear el main y el archivo .html
- Cada integrante agrega un apartado al archivo
- Cada integrante sube sus cambios a sus ramas correspondientes
- Cada integrante solicita un pull request a la rama developer para validación
- El líder valida todos los pull requst hechos a la rama de developer y es el encargo de enviarlos al main si pasa la valdiación.
